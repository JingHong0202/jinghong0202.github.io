优先级 = > ^~（匹配开头） > ~（正则） > /

反向代理

设想： 访问 http://localhost/music 代理到 http://music.jinghong.ml

写法如下：

location /music {
    proxy_pass  http://music.jinghong.ml
}
location /music/ {
    proxy_pass  http://music.jinghong.ml
}

两者不同点:  前者  实际代理的地址是 http://music.jinghong.ml/music,会把后面的参数一并带入
            后者 则相反，不会把参数带入， 代理地址 http://music.jinghong.ml
